# 業務自動化(RPA)ポートフォリオ
**統括リーダーとして主導した「月20時間の工数削減」プロジェクト**

これは、私がアルバイト先（300人規模の組織）で統括リーダー として業務DXを主導した際に、**「現場の課題」を特定し、「解決策を設計（要件定義）」し、「AIを活用して実装」**したRPAシステムのデモンストレーション・コードです。

## P：課題 (Problem)
当時、私が所属していた組織（年間予算1000万円・300人規模） では、非効率な手作業が常態化していました。

* **課題:** 「スプレッドシートの情報に基づき、PDFを手作業でページ分割・仕分けする」という単純作業。
* **損失:** この作業に**月20時間以上**の工数がかかり、予算を逼迫。
* **リスク:** 誤った仕分けによる**ヒューマンエラー**が多発し、業務品質の低下を招いていました。

## A：行動 (Action) - AIを活用した要件定義
私はこの課題を「組織の生産性を阻害する根本的な問題」と捉え、統括リーダーとして自動化システムの導入を設計しました。

プログラミング言語の知識はゼロでしたが、AI（Copilot）を「実装のパートナー」として活用し、私自身が以下の**「要件定義」**を行いました。

1.  **指示書の自動化:** 人間の「勘」ではなく、`list_demo.csv` という**指示書CSV**を読み込み、処理すべきタスクを自動で認識させる。
2.  **PDFの自動分割:** 指示書に記載されたページ数(NOP)に基づき、`PyPDF2`ライブラリを用いてPDFを正確に分割する。
3.  **PDFの自動仕分け:** 分割後のPDFと元のPDFを、指示書通りのファイル名（AfterQ, AfterE）に変更し、所定のフォルダ（`output_Q`, `output_E`）に自動で格納する。

この `portfolio_rpa_demo.py` は、機密情報を一切含まない形で、上記の**「課題解決のロジック」**だけを安全に再現したものです。

## R：結果 (Result)
このRPAシステムの導入により、**月20時間かかっていた作業工数をほぼ0**にし、ヒューマンエラーも撲滅できました。

私は単にコードを書いたのではなく、「現場の課題」を特定し、「解決策を設計（要件定義）」し、「AIを活用して実装」し、「組織に導入」するという、**DX推進の全プロセスを統括リーダーとして完遂**しました。この経験は、ITコンサルタントとしてクライアントの業務改革を推進する上で、私の核となるスキルであると確信しています。

## デモの実行方法

1.  このリポジトリをローカルにクローンまたはダウンロードします。
2.  必要なライブラリをインストールします。
    ```bash
    pip install PyPDF2 reportlab
    ```
3.  `portfolio_rpa_demo.py` を実行します。
    ```bash
    python portfolio_rpa_demo.py
    ```
4.  実行後、`input` フォルダにダミーのPDFが自動生成され、`output_Q` と `output_E` フォルダに指示書通りに仕分け・分割されたPDFが格納されていることを確認できます。

## 使用技術
* **Python**
* **PyPDF2:** PDFの分割・加工
* **reportlab:** （デモ用）ダミーPDFの自動生成
* **CSV:** 業務指示書の設計
